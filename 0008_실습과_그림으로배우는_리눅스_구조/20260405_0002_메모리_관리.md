```aiignore
               total        used        free      shared  buff/cache   available
Mem:            15Gi       1.5Gi        13Gi        12Mi       953Mi        14Gi

colaman@D3-VM-DEV-00:~$


total : 
시스템에 탑재된 전체 메모리 용량

free : 
표기상 이용하지 않는 메모리(available 필드 참조)

buff/cache : 
버퍼 캐시 또는 페이지 캐시가 이용하는 메모리이다. 시스템의 빈 메모리(free 필드값)가 부족하면 커널이 해제한다.

available : 
실질적으로 사용가능한 메모리이다. 
free 필드값의 메모리가 부족하면 해제되는 커널내의 메모리 영역 사이즈를 더한 값으로 ,
해제될 수 있는 메모리에는 버퍼캐시나 페이지 캐시의 대부분 혹은 다른 커널 내의 메모리 일부가 포함된다. 
```

### 메모리 부족 
```aiignore
메모리 사용량이 증가하면 free 가 줄어든다.
free 가 부족하면 메모리 관리 시스템은 커널 내부의 해제가능 한 메모리 영역을 해제한다.
이후에도 메모리 사용량이 증가하면, 시스템은 "Out Of Memory(OOM)" 상태가 된다.

OOM 인 경우
메모리 관리 시스템에는 적절한 프로세스를 선택해 강제 종료(kill) 하여, 
메모리 영역을 해제시키는 "OOM Killer" 라는 기능이 있다.
```


### 고수준 레벨에서의 메모리 할당
```aiignore
리눅스에서는 malloc() 함수에서 mmap() 함수를 호출하여 구현한다.

mmap() 함수는 페이지 단위로 메모리를 확보하지만,
malloc () 함수는 바이트 단위로 메모리를 확보한다.

바이트 단위로 메모리를 확보하기 위해 glibc 에서는 사전에 mmap() 시스템 콜을 이용하여, 
커다란 메모리 영역을 확보하여 메모리 풀을 만든다.
그리고, 프로그램에서 malloc() 이 호출되면 메모리 풀로부터 필요한 양을 바이트 단위로 잘라내서 반환하는 처리를 한다.
풀로 만들어둔 메모리에 더 이상 빈 공간이 없으면 다시 한번 mmap() 을 호출하여 새로운 메모리 영역을 확보한다.
```
